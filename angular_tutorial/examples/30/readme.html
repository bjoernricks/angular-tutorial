Was ist neu?
  • Bindung von Funktionen

Im vorherigen Beispiel könnte der Controller der Direktive die Personen aus der Liste entfernen. Normalerweise möchte man die an eine Direktive übergebenen Daten aber nicht durch die Direktive ändern lassen. Um der Direktive trotzdem eine Möglichkeit zu geben Aktionen auszuführen, können Callbacks verwendet werden.

Über die Notation & können Expressions an den isolierten Scope der Direktive gebunden werden. Diese Expressions sind dann als Funktion innerhalb der Direktive unter dem gebundenen Namen aufrufbar. Sollen der Expression Argumente übergeben werden, müssen diese in der Objekt Syntax ({attr: value}) notiert werden und attr muss dabei dem Namen der Variable aus der Expression entsprechen. Daher funktioniert on-delete="ctrl.remove(pal)" aus diesem Beispiel nicht.

Neben dem Binding per = und & gibt es noch eine weitere Möglichkeit. @ dient zum Binden von Daten in eine Richtung und zwar zur Direktive. D.h. Die Übergebenen Daten können von der Direktive nicht geändert werden. Ändern sich aber die Daten im aufrufenden Kontext, wird die Änderung auch in der Direktive sichtbar gemacht.

Zusammengefasst:

scope: { oneWayBinding: '@', twoWayBinding: '=', expression: '&' }
