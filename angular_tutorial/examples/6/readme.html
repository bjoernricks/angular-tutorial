Modul

  • Ein Modul enhält Komponenten einer Angular Webanwendung.
  • Ein Modul dient somit zur logischen Strukturierung eine Webanwendung.
  • Über das ng-app Attribut wird das zu ladende Modul angegeben.
  • In einem Modul können beliebig viele Controller definiert werden.

Controller und Scope

  • Anwendungslogik wird einem Controller gekapselt.
  • Jeder Controller bildet einen eigenen Namensraum.
  • Der Namensraum in AngularJS wird als Scope bezeichnet und durch die Variable $scope repräsentiert.
  • Wird in einem Angular Template kein Controller angegeben, referenzieren die Variablen den $rootScope, wie das implizit in den vorherigen Beispielen der Fall war.
  • Wird in einem Angular Template ein Controller angegeben, referenzieren die Variablen den $scope des Controllers.
  • Der Controller kann an jedes beliebige HTML Element gesetzt und auch in einem Kind-Element überschrieben werden.
