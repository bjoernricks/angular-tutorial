Was ist neu?
  • .module('nameApp', ['ngRoute'])
  • Aufteilung der Anwendung in mehrere Module
  • .config zum Definieren der Routen
  • ng-view statt ng-controller
  • Anwendung besteht aus mehreren Templates
  • Templates werden je nach URL geladen und verwendet

Unsere Anwendung hängt jetzt von ngRoute ab. Daher muss die entsprechende JavaScript Datei angegeben und geladen werden. Ebenfalls muss unser Anwendungsmodul die Abghängigkeit explizit definieren .module('nameApp', ['ngRoute']).

Weiterhin ist unsere Anwendung nun auf mehrere Module (nameApp und nameApp.controllers) in entsprechenden JavaScript Dateien augeteilt. Da die Controller für die Routendefinition im Haupt-Modul verwendet werden, hängt das Haupt-Modul auch von nameApp.controllers ab.

Statt der ngController Direktive wird jetzt ngView verwendet. Die Haupt-HTML-Datei (hier index.html) definiert nur noch die Grundstruktur. Je nach Navigationsziel des Nutzers wird ein Controller und eine View (Template) geladen und an die Stelle von ng-view gesetzt.

Die Routen müssen über eine Config Funktion definiert werden. Config Funktionen werden zum Start der Anwendung von Angular JS aufgerufen, noch bevor Services initialisiert werden. Daher können auch keine Services in die Config Funktionen injected werden. Stattdessen müssen hier Provider verwendet werden, die wiederum die Services konfigurieren können.

Über den $routeParams Service können die Routenparameter in den Controllern abgerufen werden.
