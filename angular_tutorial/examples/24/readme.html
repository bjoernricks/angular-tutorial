Was ist neu?
  • Es wird eine Directivenfunktion als name registriert
  • Eine Direktivenfunktion muss ein sogenanntes Directive Definition Object zurückgeben
  • Das Directive Definition Object bestimmt die Funktionsweise der Direktive

Über Direktiven kann HTML um weiteres Verhalten und Aussehen erweitert werden. Sie bilden somit wiederverwendbare UI Elemente.

Die Direktiven werden aus ausgeführt, wenn deren registrierter Name (nach einer bestimmten Normalisierung siehe Directive Guide) im HTML Code gefunden wird. Im einfachsten Fall wird anschließend die Direktive durch neuen HTML Code aus einem Template ersetzt.

Über die restrict Property des Directive Definition Objects kann dabei angegeben werden, welcher HTML Code dabei durchsucht wird.

  • 'A' - HTML Attributenamen werden berücksichtigt
  • 'E' - HTML Elementnamen werden berücksichtigt
  • 'C' - HTML CSS Klassen werden berücksichtigt
  • 'M' - HTML Kommentare werden berücksichtigt
Diese Einschränkungen können kombiniert werden. 'AEC' berücksichtigt dann z.B. Attributnamen, Elementnamen und CSS Klassen. Wird die restrict Property nicht explizit angegeben, gilt der Default-Wert 'AE'.
