Was ist neu?
  • Über .factory können Services registriert werden
  • Services können Daten und Methoden kapseln

Ein Service kapselt Daten in einer abgeschlossenen Komponente. D.h. die Datenhaltung sollte immer über einen Service realisiert werden.

Services sind Singletons. D.h. sie werden nur einmal pro Angular JS Anwendung initalisiert. Somit kann über über sie auch anwendungsweite Logik und Daten geteilt werden.

Services können den direkten Zugriff auf Daten einschränken. Im Beispiel ist kein Zugriff auf NameService.names im Controller möglich.

Services können von anderen Services abhängig sein. Z.B.:

namesService.$inject = ['$log']; function namesService($log) { var names = ['Larry', 'Curly', 'Moe', 'John', 'Kelly', 'Thomas']; var service = { getNames: get_names, }; return service; function get_names() { $log.debug('Returning all names ', names); return names; } }
