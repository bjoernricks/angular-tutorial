  • Controller As Syntax bei Direktiven

Die Controller As Syntax kann ebenfalls bei Controllern für Direktiven verwendet werden.

Über die controllerAs Property im Directive Definition Object kann der Name des Controllers im Scope gesetzt werden. So kann festgelegt werden, unter welcher Variable der Controller im Template angesprochen wird.

Für das Binden der Daten aus dem aufrufenden Code direkt an den Controller, muss eine weitere Property gesetzt werden. Und zwar bindToController. Diese Property funktioniert seit Angular 1.4 genauso wie die scope Property, nur dass die Daten nicht an den Scope sondern an den Controller gebunden werden.

Eine gute Anleitung in weitere Hinweise zur Controller As Syntax gib es unter Exploring Angular 1.3: Binding to Directive Controllers. Eine weitere sehr gute Anleitung (allerdings geschrieben bevor die Controller As Syntax eingeführt wurde) findet sich unter The Hitchhiker’s Guide to the Directive.
